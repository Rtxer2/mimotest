# i18n 多语言支持设计规格

## [S1] 问题

系统界面语言混合中英文，且不支持切换。需要添加中文、英文、西班牙语三套语言支持。

## [S2] 解决方案

使用 react-i18next 库，创建翻译文件，将所有硬编码文本替换为翻译 key。在右上角添加语言切换器。语言偏好存 localStorage。

## [S3] 技术选型

- 库: `react-i18next` + `i18next` + `i18next-browser-languagedetector`
- 存储: localStorage (`i18nextLng`)
- 语言: `zh` (中文), `en` (English), `es` (Español)
- 默认语言: `zh`

## [S4] 翻译文件结构

```
frontend/src/i18n/
  index.ts
  locales/
    zh.json
    en.json
    es.json
```

翻译 key 按模块命名空间分组：`sidebar.*`, `dashboard.*`, `orders.*`, `customers.*`, `production.*`, `inventory.*`, `quality.*`, `approvals.*`, `notifications.*`, `system.*`, `common.*`, `login.*`, `analytics.*`

## [S5] 语言切换器

- 位置: Layout header 右上角
- 组件: antd Select，选项为 3 种语言
- 切换后立即生效，无需刷新

## [S6] 组件改造

所有页面组件使用 `useTranslation()` hook 获取 `t` 函数，替换硬编码文本。antd 组件的内置文本通过 `ConfigProvider` 的 `locale` 属性切换。

## [S7] 范围

29 个 TSX 文件需要改造：
- 框架: Layout, Sidebar, ProtectedRoute, Login, App
- 页面: Dashboard, AnalyticsDashboard, 各模块页面
- 组件: NotificationBell
